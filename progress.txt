## Task: Implement YouTube Data API v3 integration

### Changes Made:
- Created YouTube API client with OAuth 2.0 flow and video upload functionality
- Added platform connections routes for OAuth authentication and token management
- Updated clips bulk export endpoint to support YouTube video uploads
- Created database migration for platform_connections table
- Added comprehensive tests for YouTube integration

### Files Created:
- server/src/lib/youtube.ts - YouTube API client with OAuth and upload methods
- server/src/routes/platforms.ts - Platform connection routes with OAuth flow
- server/migrations/002_create_platform_connections_table.sql - Database schema
- server/src/lib/__tests__/youtube.test.ts - YouTube API tests (10 tests passing)
- server/src/routes/__tests__/platforms.test.ts - Platform routes tests

### Files Modified:
- server/src/lib/env.ts - Added YOUTUBE_CLIENT_ID, YOUTUBE_CLIENT_SECRET, YOUTUBE_REDIRECT_URI
- server/src/routes/clips.ts - Updated bulk export endpoint to upload to YouTube
- server/src/index.ts - Mounted platforms routes at /api/platforms

### Features Implemented:
1. OAuth 2.0 flow with Google for YouTube access
2. Token refresh mechanism for long-lived connections
3. Video upload to YouTube with metadata (title, description, tags, privacy status)
4. Automatic #Shorts tag for YouTube Shorts
5. Gaming category (ID: 20) set by default
6. Quota error handling (10,000 units/day limit, 1,600 per upload)
7. Platform connection management (connect, disconnect, list)
8. Secure token storage in database with RLS policies

### API Endpoints:
- GET /api/platforms/connections - List connected platforms
- GET /api/platforms/youtube/connect - Initiate OAuth flow
- GET /api/platforms/youtube/callback - Handle OAuth callback
- DELETE /api/platforms/:platform/disconnect - Revoke platform connection
- POST /api/clips/bulk/export - Export clips to platforms (YouTube implemented)

### Database Schema:
- platform_connections table with columns:
  - id, user_id, platform, access_token, refresh_token
  - token_expires_at, username, platform_user_id, metadata
  - connected_at, updated_at
  - Unique constraint on (user_id, platform)
  - RLS enabled for user isolation
