## Task: Add hook detection and clip boundary optimization

### Implementation Summary:
Implemented a comprehensive hook detection and clip boundary optimization system to improve clip quality and viewer retention by ensuring clips start with strong hooks and end on complete thoughts.

### Changes Made:

#### 1. Created Hook Detection Module (server/src/analysis/hooks.ts)
- **Hook Analysis Function (`analyzeHook`)**: Analyzes the first 3 seconds of clips to detect hook quality
  - Detects 4 types of strong hooks:
    - **Question hooks**: Starts with "What", "How", "Why", etc. (base score: 85)
    - **Reaction hooks**: Starts with "Oh my god", "No way", "Wait", etc. (base score: 90)
    - **Action hooks**: Starts with "Watch", "Check this", "Look", etc. (base score: 80)
    - **Statement hooks**: Declarative sentence with capitalization (base score: 70)
  - Applies penalties for weak starts:
    - Mid-sentence starts: -30% penalty
    - Silent starts (>0.5s delay): -40% penalty
    - Filler word starts (um, uh, like): -20% penalty
  - Bonus for high word density (confident delivery): +10%

#### 2. Clip Boundary Optimization Function (`optimizeClipBoundaries`)
- **Start Boundary Optimization**: Aligns clip start to sentence boundaries
  - Searches backward up to 2 seconds to find sentence start
  - Avoids starting with filler words
  - Ensures clip captures full context
- **End Boundary Optimization**: Ensures clips end on complete thoughts
  - Aligns to sentence boundaries
  - Adds 0.5s breathing room after last word
  - Prevents mid-sentence cuts
- **Duration Enforcement**: Ensures clips meet minimum 8-second duration

#### 3. Hook Scoring Integration
- **`applyHookScore` function**: Applies hook score to base clip score
  - Default weight: 20% (configurable)
  - Boosts clips with strong hooks
  - Formula: baseScore + (hookScore * hookWeight)
- **`filterWeakHooks` function**: Filters out clips with poor hooks
  - Default threshold: hookScore < 30 are filtered
  - Configurable threshold for different quality standards

#### 4. Updated Fusion Types (server/src/analysis/fusion.ts)
- Extended `SignalMoment` interface to include:
  - `hookScore?: number` - Hook quality score (0-100)
  - `boundaryOptimization?` - Adjusted clip boundaries with reasoning

### Integration Points:

The hook detection system is designed to integrate into the pipeline at these points:

1. **During Caption Stage**: After transcription is complete, run `analyzeHook()` on each clip
2. **In Fusion Stage**: Apply hook scores to moments using `applyHookScore()`
3. **Before Extraction**: Optimize boundaries with `optimizeClipBoundaries()`
4. **Final Filtering**: Remove weak hooks with `filterWeakHooks()`

### Configuration Options:

The system provides several configurable parameters:
- `hookWeight`: Impact of hook score on final score (default: 0.2)
- `minHookScore`: Threshold for filtering weak hooks (default: 30)
- `maxAdjustment`: Maximum seconds to adjust boundaries (default: 2)
- `hookWindow`: Seconds analyzed for hook detection (default: 3)

## Task: Add multi-face handling and speaker prioritization

### Changes Made:
- Enhanced server/src/extraction/reframe.ts with comprehensive multi-face handling and speaker prioritization
- Added new interfaces: DetectedFace, FaceTrack, FaceSelectionStrategy
- Extended ReframeConfig to support faceStrategy and audioMoments parameters
- Implemented IOU-based face tracking for consistent face identification across frames
- Added three face selection strategies:
  1. 'largest': Selects face with largest average bounding box area
  2. 'speaker': Correlates faces with audio moments for speaker detection
  3. 'center': Selects face closest to frame center
- Implemented speaker prioritization using audio moment correlation (within 0.5s window)
- Added face persistence tracking to avoid jumping between different faces
- Created comprehensive test suite in server/src/extraction/reframe.test.ts with 19 test cases

- Speaker track selection tests (audio correlation, proximity weighting, fallback)
- Center track selection tests
- Integration tests for multi-face handling scenarios

### Implementation Details:
1. **Face Tracking**: Uses Intersection Over Union (IOU) algorithm with 0.3 threshold
2. **Temporal Continuity**: Only tracks faces within 1 second of each other
3. **Speaker Detection**: Correlates faces with audio moments within 0.5s window
4. **Proximity Weighting**: Linear weighting based on temporal distance to audio peaks
5. **Graceful Fallback**: Falls back to largest face when no audio correlation exists
