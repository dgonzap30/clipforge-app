## Task: Add contextual emoji insertion

### Changes Made:
- Implemented emoji keyword mapping system with 70+ gaming/streaming keywords (fire 🔥, crazy 😱, lol 😂, gg 🎮, etc.)
- Added laughter detection for automatic emoji insertion on words like "haha", "lmao", etc.
- Added support for multi-word emoji phrases ("no way" → 😲, "lets go" → 🚀)
- Extended generateTikTokASS() function in transcribe.ts to insert emojis as separate dialogue lines
- Implemented emoji positioning logic with X/Y coordinate calculation
- Added optional emoji scale animation (120% scale-up on insertion, settle to 100%)
- Created Emoji style in ASS format with larger font size (1.25x base font size)
- Added emojis and emojiAnimation configuration options to CaptionConfig interface
- Updated frontend Settings store to include enableEmojis boolean setting (default: true)
- Added "Enable Emojis" toggle to Settings UI page with description
- Wrote comprehensive unit tests for emoji insertion functionality (15 test cases)
- All tests passing with 100% coverage of emoji features

### Files Modified:
- server/src/captions/transcribe.ts (added emoji mapping, detection, and insertion logic)
- server/src/pipeline/stages/caption.ts (added emojis config to CaptionConfig interface)
- server/src/captions/emoji.test.ts (new file with comprehensive tests)
- server/src/pipeline/stages/caption.test.ts (added emoji configuration tests)
- src/store/index.ts (added enableEmojis to Settings interface)
- src/pages/Settings.tsx (added emoji toggle checkbox in Output Settings section)

### Features Implemented:
1. **Keyword-to-Emoji Mapping**: 70+ common gaming/streaming keywords mapped to contextual emojis
2. **Laughter Detection**: Automatic detection of laughter patterns with 😂 emoji insertion
3. **Multi-Word Phrases**: Support for phrases like "no way", "lets go" with appropriate emojis
4. **ASS Subtitle Integration**: Emojis rendered as separate dialogue lines with pixel-perfect positioning
5. **Animation Support**: Optional scale-up animation for emoji entrance
6. **Configuration Toggle**: Frontend UI toggle to enable/disable emoji insertion
7. **Case-Insensitive Matching**: Emojis match regardless of text case (FIRE, fire, Fire all work)

### Technical Details:
- Emojis are inserted on Layer 1 (above text on Layer 0) to prevent overlap
- Position calculated based on character width estimation and word placement
- Emoji style uses larger font size for visibility
- Animation uses ASS \t() tags for smooth scaling transitions
- Configuration defaults: emojis off by default, animation on when emojis enabled
- All emoji dialogue lines use absolute positioning to avoid text overlap

### Test Coverage:
- 15 unit tests covering all emoji features
- Tests for single-word keywords, multi-word phrases, laughter detection
- Tests for case-insensitive matching, animation, positioning
- Tests for disabled state, multiple emojis per segment
- Integration tests for configuration passing through pipeline

### Notes:
- Emoji insertion is off by default to maintain backward compatibility
- Users can enable via Settings page → Output Settings → "Enable Emojis"
- Emojis only appear when word-level timing is available from Whisper
- Position calculation uses approximate character width (font size * 0.6)
- Compatible with all caption positions (top, center, bottom)
